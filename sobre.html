Nova Lima GIS — POC v1

Prova de Conceito de uma aplicação SIG municipal desenvolvida com JavaScript puro e ArcGIS Maps SDK for JavaScript.

Voltar para o mapa

Visão geral

Esta aplicação representa uma primeira versão funcional de um Sistema de Informação Geográfica voltado ao município de Nova Lima/MG.

O objetivo da POC v1 foi conhecer e explorar conceitos fundamentais de SIG (Sistema de Informação Geográfica) construindo uma base simples, organizada e extensível para visualização de dados geográficos. JavaScript do ArcGIS.

Objetivo da POC

Mais do que criar um mapa interativo, esta versão foi desenvolvida como um exercício de aprendizado progressivo. Cada recurso foi adicionado de forma incremental, sempre buscando compreender o motivo técnico e arquitetural de cada decisão.

A aplicação serve como base para evolução futura em direção a um SIG municipal mais completo, com camadas reais, análises espaciais, filtros, serviços ArcGIS REST e integração com dados públicos.

Conceitos de SIG explorados

  • Representação vetorial de dados geográficos.
  • Geometrias do tipo Point, LineString e Polygon.
  • Camadas geográficas temáticas.
  • Feições geográficas e seus atributos.
  • Uso do formato GeoJSON.
  • Sistema de referência espacial WGS84 / WKID 4326.
  • Seleção visual de feições no mapa.
  • Consulta espacial com geometryEngine.contains().
  • Relação espacial entre bairros, escolas e UBS.
  • Representação de rede viária com geometrias lineares.

Arquitetura da aplicação

A aplicação foi organizada com separação clara de responsabilidades, evitando concentrar toda a lógica em um único arquivo.

app.js

Responsável pela inicialização do mapa, criação da view, widgets, eventos principais e orquestração da aplicação.

camadas.js

Funciona como catálogo de camadas da aplicação, concentrando simbologia, fontes GeoJSON, títulos e templates de exibição.

gis.js

Contém funções de integração com o ArcGIS, como carregamento de GeoJSON, criação de Graphics e controle de seleção.

clickController.js

Centraliza a lógica de clique no mapa, identificando a camada selecionada e atualizando o painel lateral.

services.js

Responsável por integrações externas, como a consulta à API pública do IBGE.

helpers.js

Reúne funções auxiliares de interface, como montagem de listas e pequenos componentes HTML.

Camadas implementadas

  • Limite municipal: carregado a partir da API pública do IBGE.
  • Bairros: camada local mockada com geometria Polygon.

  • UBS: camada local mockada com geometria Point.
  • Rede viária: camada local mockada com geometria LineString.

Funcionalidades da versão 1

  • Visualização do município de Nova Lima/MG.
  • Carregamento de camadas locais em GeoJSON.
  • Exibição de bairros, escolas, UBS e ruas.
  • Seleção visual de feições.
  • Painel lateral customizado para informações.
  • Desativação do popup padrão do ArcGIS.
  • Consulta de escolas e UBS contidas em cada bairro.
  • Uso de widgets nativos do ArcGIS.
  • Controle de camadas com LayerList.
  • Legenda automática com Legend.
  • Busca geográfica com Search Widget.

Aprendizados principais

  • Como estruturar uma aplicação SIG sem framework.
  • Como separar responsabilidades entre arquivos JavaScript.
  • Como representar diferentes geometrias no ArcGIS.
  • Como transformar GeoJSON em Graphics do ArcGIS.
  • Como trabalhar com seleção e destaque de feições.
  • Como organizar um catálogo de camadas reutilizável.
  • Como integrar dados públicos do IBGE.
  • Como pensar a evolução incremental de uma aplicação GIS.

Limitações da versão 1

  • As camadas locais ainda são mockadas.
  • A seleção ainda utiliza estado global provisório.
  • As consultas espaciais ainda são simples.
  • A aplicação ainda não usa FeatureServer.
  • Não há persistência de dados.
  • A rede viária ainda não possui análise espacial avançada.
